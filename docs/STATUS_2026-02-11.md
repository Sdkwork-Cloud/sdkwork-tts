# Status Update - February 11, 2026

## Bottom line

The Rust pipeline now produces intelligible speech on GPU. The previous dominant failure mode (low-frequency rumble / water-like output) is no longer the primary output in current runs.

## Completed today

- GPT LM-head norm parity fix:
  - Added separate LM-head norm (`final_norm.weight`, `final_norm.bias`) instead of reusing transformer `ln_f`.
- GPT multi-step parity instrumentation:
  - Added Rust dump fields for step1 and last-step logits + cache length growth.
  - Extended Python comparer to replay cached decode and compare per-step logits/cache growth.
- DiT parity alignment updates (already integrated in code by end of session):
  - RMSNorm Ada norms, RoPE, UViT skip schedule, adaptive final norm behavior.
- BigVGAN correction:
  - Replaced fake repeat+conv upsampling with true transposed convolution.
  - Removed non-upstream extra Snake activation before upsample stage.
- Inference guardrail:
  - Added explicit `--de-rumble` high-pass and auto fallback on high DC bias.
